The Four Chunks

Chunk 1: Building the Front End

Static Website

~15 hours

In this chunk, participants build the frontend of their cloud-hosted resume, deploying it to a live URL for sharing. Steps include writing the resume in HTML, styling it with CSS, deploying it as a static website on Amazon S3, ensuring HTTPS security with Amazon CloudFront, and pointing a custom DNS domain name to the CloudFront distribution.

Chunk 2: Building the API

Visit Counter

~10 hours

In this chunk, participants develop a serverless API on AWS that updates a visitor counter in a database. Using DynamoDB for data storage and AWS API Gateway with Lambda functions for API creation, they establish a robust backend infrastructure.

Chunk 3: Front-end / Back-end Integration

Testing

~15 hours

In this section, participants merge the front-end and back-end components of their Cloud Resume Challenge project. They connect the resume website to the serverless API, enabling the display of the visitor counter on their homepage. Additionally, automated tests help ensuring the functionality of the code.

Chunk 4: Automation / CI

CI/CD Pipelines

~20 hours

In this section, participants focus on automating the deployment process and implementing continuous integration (CI) for their Cloud Resume Challenge project. They define their cloud resources using infrastructure-as-code (IaC) principles, utilizing CLoudFormation templates for backend resources. Finally, participants set up CI/CD pipelines using GitHub Actions, ensuring that updates to the codebase trigger automated testing and deployment processes for both backend and frontend components.

About

It was introduced by Forrest Brazeal on Twitter and LinkedIn in 2020. in 16 steps, participants deploy a personal website on cloud platforms like AWS. It is the best practical way to gain hands-on experience with cloud services.
