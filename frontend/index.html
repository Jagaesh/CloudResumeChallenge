The Four Chunks

Chunk 1: Building the Front End

Static Website

~15 hours

In this chunk, participants build the frontend of their cloud-hosted resume. Steps include writing the resume in HTML, styling it with CSS, deploying it as a static website on Amazon S3, ensuring HTTPS security with Amazon CloudFront, and pointing a custom DNS domain name to the CloudFront distribution.

Chunk 2: Building the API

API

~10 hours

The goal here is to develop a serverless API on AWS that updates a visit counter in a database. Services used are DynamoDB for data storage, AWS API Gateway and Lambda functions.

Chunk 3: Front-end / Back-end Integration

Testing

~15 hours

Participants will now connect the resume website to the serverless API, enabling the display of the visit counter on the homepage. Additionally, automated tests help ensuring that the code is valid.

Chunk 4: Automation / CI

Pipelines

~20 hours

Finally, it is time to define our cloud resources using CloudFormation templates. Moreover, for both backend and frontend, CI/CD pipelines in GitHub Actions will automate testing and deployment after each update to the code.

About

The Cloud Resume Challenge is an opportunity to demonstrate cloud computing skills. It was introduced by Forrest Brazeal on Twitter and LinkedIn in 2020. in 16 steps, participants deploy a personal website on cloud platforms like AWS. It is the best practical way to gain hands-on experience with cloud services.
